# Raid-Helper 대비 한길련봇 — 완전 우위 체크리스트 (디스코드 봇 한정)

> 비교 범위: **디스코드 안에서의 기능만.** (웹은 우리가 무조건 우위라 비교 제외)
> 한길련봇 상태: ✅완료 · 🟡계획(설계됨) · ❌없음(추가 필요)
> 목표: 아래 표의 **모든 행이 "동등" 또는 "우위"** 가 되게 만든다. ❌가 하나라도 남으면 빈틈.

| 역할/클래스/특성 선택 | ✅ 매번 선택 | 🟡 저장된 캐릭터에서 | **우위** | 캐릭터 영구저장 → 재선택 불필요 |

## C. 한길련봇만의 우위 (Raid-Helper엔 없음)

| 우위 기능 | 설명 |
|---|---|
| **웹 ↔ 디스코드 완전 실시간 통합** | 같은 DB. 어디서 하든 즉시 양쪽 반영. Raid-Helper는 디스코드 단독. |
| **연합(다길드) 개념** | 연합 + 길드별 + 외부참가 허용 필터. Raid-Helper엔 없음. |
| **영구 캐릭터 프로필** | 클래스/특성/스왑/아이템레벨 저장 → 신청 시 재입력 0. |
| **벤치(예비전력) 시스템** | 예비 인원 별도 관리. |
| **스왑 가능 역할** | "탱스왑 가능" 등 유연 편성. |
| **시너지/구성 깊이** | WoW 한국 길드 실전 편성. |
| **난이도 인지 정원** | 신화20·일반/영웅30 자동. |
| **한국어 네이티브 + KR 서버** | 정서·용어·서버 완전 현지화. |
| **신원 통합(웹=디스코드)** | OAuth 연동으로 하나의 신원. |

## D. "무조건 이기기" 위해 추가/수정할 것 (우선순위)

### 🥇 1순위 — 핵심 갭
1. **`/레이드생성`** (디스코드에서 레이드 생성, 관리자) — Raid-Helper 간판 기능. 없으면 "디스코드 완결" 깨짐.
2. **`/신청` · `/내신청`(수정/취소)** — 진행 중 계획, 최우선.
3. **타임존 자동 표시** — 모든 시간 출력을 디스코드 `<t:unix:F>` 타임스탬프로 → 공짜 현지시간 변환.
4. **신청 마감/잠금 토글** (`/관리`).

### 🥈 2순위 — 자동화 우위
5. **반복 이벤트** (매주 같은 레이드 자동 생성).
6. **출석 통계 `/출석`** (기존 로그 집계 → 동등+).
7. **자동 임시역할** (신청자에게 이벤트 역할 → 종료 후 제거).
8. **레이드 템플릿/프리셋**.

### 🥉 3순위 — 디테일
9. 다중 신청 상태 확장(지각/미정/불참 검토).
10. 멘션 옵션, 투표(후순위), CSV(웹 대체).

## E. 판정 요약
- **이미 우위:** 알림 품질, 리마인더 촘촘함, 영구 캐릭터, 연합, 웹통합, 난이도 정원, 한국화.
- **구현하면 동등+:** 신청/수정/취소, 출석통계, 멘션.
- **반드시 메울 갭:** 디스코드 레이드 생성, 타임존 자동표시, 반복 이벤트, 신청잠금, 자동 임시역할, 템플릿.
- 위 갭만 다 채우면 **디스코드 기능에서도 Raid-Helper를 빈틈없이 추월.** + C의 우위 = "압도".

## F. 다음 작업 순서 제안
현재 `/일정` 완료. 이어서:
`/연동` → `/신청`·`/내신청` → `/레이드생성` → 타임존 타임스탬프 전환 → `/출석`·반복·자동임시역할.

이 문서를 기준선으로, 매 기능 구현 시 위 표 판정을 "동등→우위"로 갱신하며 진행한다.
