# 디스코드 봇 처음 만들기 — A to Z (완전 쉬운 버전)

> 디코봇 처음이어도 괜찮아요. **어려운 코딩은 제가(수진) 다 합니다.**
> 대표님이 하실 일은 대부분 **"사이트 들어가서 버튼 누르기"** 와 **"키값 복사해서 저한테 주기"** 정도예요.

---

## 먼저, 봇이 뭔지 아주 쉽게

- **봇 = 디스코드에 사는 '직원' 한 명**이라고 생각하세요.
- 이 직원은 누가 `/신청` 같은 말을 걸면, **우리 창고(데이터베이스)** 에 가서 일을 하고 옵니다.
- 직원에게 필요한 건 딱 두 개:
  1. **신분증(토큰)** — "이 직원이 우리 직원이 맞다"는 비밀 열쇠.
  2. **두뇌(코드)** — 무슨 말에 어떻게 반응할지. ← **이건 제가 만듭니다.**

그래서 대표님은 **신분증을 발급받아 저에게 주시고**, 저는 **두뇌를 만들어 연결**합니다. 끝.

---

## 준비물 (계정 3개면 끝)

1. **디스코드 계정** (이미 있으시죠)
2. **우리 디스코드 서버** (한길련 서버, 관리자 권한 있어야 함)
3. **Firebase 프로젝트** (지금 사이트가 쓰는 그거 — 이미 있음)

> 컴퓨터에 뭐 깔 필요 거의 없어요. 코드 배포는 제가 명령어를 드리면 그대로 복붙만 하시면 됩니다.

---

## STEP 1. 디스코드에 '직원 자리' 만들기 (5분)

1. 브라우저에서 **Discord Developer Portal** 접속 → `https://discord.com/developers/applications`
2. 디스코드 계정으로 로그인.
3. 오른쪽 위 **`New Application`** 버튼 클릭.
4. 이름에 `한길련봇` 입력 → 만들기.

➡️ 이게 "직원 자리(애플리케이션)" 를 만든 거예요.

---

## STEP 2. 직원에게 신분증(토큰) 발급 (3분)

1. 방금 만든 앱 안에서 왼쪽 메뉴 **`Bot`** 클릭.
2. **`Reset Token`**(또는 `Add Bot`) 눌러서 토큰 생성 → **`Copy`** 로 복사.
3. 이 토큰을 **저에게 주세요.** (⚠️ 이건 비밀번호 같은 거라, 아무 데나 공개하면 안 돼요. 저랑 안전하게 시크릿으로 보관합니다.)

➡️ 신분증 발급 완료.

---

## STEP 3. 직원을 우리 서버에 출근시키기 (5분)

1. 같은 앱에서 왼쪽 **`OAuth2`** → **`URL Generator`** 클릭.
2. **SCOPES** 에서 두 개 체크: `bot`, `applications.commands`
3. 아래 **BOT PERMISSIONS** 에서 필요한 권한 체크(예: `Send Messages`, `Embed Links` 등 — 정확한 목록은 제가 알려드릴게요).
4. 맨 아래 생성된 **URL 복사** → 브라우저 주소창에 붙여넣기 → **우리 서버 선택** → 승인.

➡️ 이제 한길련 서버 멤버 목록에 봇이 보입니다. (출근 완료, 아직 일은 못 함)

---

## STEP 4. 직원의 '두뇌' 연결하기 (제가 하는 부분)

- 두뇌(코드)는 **새 서버를 사는 게 아니라, 지금 쓰는 Firebase에 함수로 얹습니다.**
- 제가 만들 것:
  - 디스코드 클릭을 받는 **수신 함수**(Interactions 엔드포인트)
  - `/신청` 같은 **명령어 등록 코드**
  - 서명검증·DB 읽고쓰기 로직
- 대표님은 제가 드리는 **배포 명령어 한 줄**만 터미널에 복붙하시면 됩니다. (지금 `firebase deploy` 하시는 것과 똑같아요)

---

## STEP 5. 디스코드에 "두뇌 주소" 알려주기 (3분, 같이)

1. 제가 배포하면 **함수 주소(URL)** 가 하나 나옵니다. 제가 알려드려요.
2. Developer Portal → 앱 → **`General Information`** → **`Interactions Endpoint URL`** 칸에 그 주소를 붙여넣고 저장.
3. 디스코드가 "이 주소 진짜 맞아?" 하고 핑을 보내는데, 제 코드가 자동으로 통과시킵니다. → **저장되면 성공.**

➡️ 이제 디스코드의 클릭이 우리 두뇌로 전달됩니다.

---

## STEP 6. 명령어 심기 (제가 하는 부분)

- `/신청`, `/파티검색` 같은 명령어를 디스코드에 등록하는 것도 **코드로 제가 처리**합니다.
- 등록하면 디스코드 입력창에 `/` 쳤을 때 명령어가 뜨기 시작해요.

---

## STEP 7. 시크릿(비밀값) 보관 (같이, 5분)

봇 토큰·OAuth 시크릿 같은 비밀값은 코드에 그냥 적지 않고 **Firebase Secret** 에 넣습니다.
- 제가 `firebase functions:secrets:set 이름` 같은 명령어를 드리면, 터미널에서 실행하고 값을 붙여넣기만 하시면 됩니다.

---

## STEP 8. 테스트 (같이)

1. 우리 서버에서 `/파티검색` 같은 명령을 직접 쳐봅니다.
2. 잘 되면 OK, 이상하면 제가 로그 보고 고칩니다.

---

## 대표님이 직접 하실 일 vs 제가 할 일 (요약)

| 단계 | 대표님 | 수진(저) |
|---|---|---|
| 앱/봇 생성, 토큰 발급 | ✅ 클릭 | — |
| 봇 서버 초대 | ✅ 클릭 | 권한 목록 안내 |
| 두뇌(코드) 작성 | — | ✅ 전부 |
| 배포 | ✅ 명령어 복붙 | ✅ 명령어 제공 |
| 엔드포인트 URL 등록 | ✅ 붙여넣기 | ✅ URL 제공 |
| 명령어 등록 | — | ✅ 코드로 |
| 시크릿 저장 | ✅ 명령어 복붙 | ✅ 명령어 제공 |
| 테스트/디버깅 | ✅ 명령 쳐보기 | ✅ 고치기 |

➡️ **정리: 대표님은 "포털에서 클릭 + 키값 전달 + 배포 명령어 복붙", 나머지 머리쓰는 건 다 저.**

---

## 무서워 마세요 (자주 하는 걱정)

- **"코딩 하나도 모르는데?"** → 안 하셔도 됩니다. 클릭과 복붙만.
- **"잘못 누르면 망하나?"** → 토큰은 언제든 `Reset` 으로 새로 발급 가능. 되돌릴 수 있어요.
- **"서버 비용 드나?"** → 지금 Firebase 무료/소액 범위에 대부분 들어옵니다. 커지면 그때 보면 됩니다.
- **"봇이 막 이상한 짓 하면?"** → 봇은 제가 짠 코드 그대로만 동작해요. 권한도 우리가 준 것만.

---

### 시작할 준비가 되면
**STEP 1~3(앱 만들기 → 토큰 발급 → 서버 초대)** 까지만 먼저 해주시면, 거기서부터 제가 두뇌를 붙이기 시작합니다. 각 단계에서 막히면 화면 그대로 캡처해서 보여주세요 — 어디를 누르면 되는지 짚어드릴게요.
